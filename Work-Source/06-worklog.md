* 公众号，小程序与APP的优缺点对比
  ```

      首先，我们搞清楚为什么要做公众号，小程序与APP？做三个终端的目的是为什么？
    我以为做公众号，小程序与APP的目的，是延申我们产品的使用长度，并且在这个延长范围内更好的
    增加用户黏度，强化用户粘性，从这个角度考虑，我们结合成本问题分析。

    分析关注点：是否能够延申产品使用时间长度，能否更好的增加用户粘性。

  ```
* 公众号+小程序”将成标配打法
  ```
    第一个是组合原则。
          一个主体认证50个小程序。50个小程序的界面切换和底层数据互联互通，无缝跳转。少即是多、多即是少；分久必合、合久必分。
          一个企业不会开50个公众号，也不会去建50个网站，但它一定会去做50个小程序，并且小程序之间是通的，页面可以互相跳转，也即小程序的组合。

    第二个是社交原则。
          小程序基于微信生态，必须构建在微信生态之中，否则就是空中楼阁，如果某个项目的创始人没有太多微信生态的背景就做小程序，我认为这个企业不值得投。
          如何与微信流量对接？做好微信社交是关键。
          有一款名为“包你说”的小程序，它的设计很简单，用户A在朋友圈发一个红包，用户B读一遍就可以拿到红包，然后平台收一点手续费，完全是C2C的打法，现在的流水和利润非常可观。

    第三个是产品原则。
          小程序是生产工具。生产工具的目的是要生产出产品。小程序要搭建出消费场景，绝大多数的消费体验，可以在小程序里完成。
          广东省刚刚上线一款名为“粤省事”的小程序，社保、公积金、交通、出入境签证等所有办事流程都可以在上面非常轻巧地完成，非常方便。

    第四个是简洁原则。一个功能一个小程序。用完即走、下次再来。

    第五个是高效原则。所思即所见，所见即所得。随时更改、随时调整。如果不动框架的话，UI方面怎么改都行，现在还可以远程测试。

    第六个是数据原则。
        小程序提供精准用户画像、用户留存，由于目前在主动推送方面还有限制，所以公众号+小程序，是目前解决数据管理的优选方案。

    第七个是开放原则。
        不同主体的小程序之间可以关联，一个小程序可以关联500个公众号，可以关联500个APP。
        没有开放的心态，应用不好小程序。以前打开公众号上的一篇文章，用户不好跳出去，
        所以在没有小程序之前，电商找我们合作都拒绝了。

小程序塑造新商业，三个条件成必要
公众号改变了中国媒体，小程序野心更大，想彻底改变中国商业。

  ```

* 对比我们中外企业管理
  ```
      对比我们中外企业管理，我们目前着重推崇西方管理系统，正因为西方管理系统已经体现出了优越的管理效果，
    备受国人推崇，又其科学系统的管理方法所体现。在国内，宣传科学，推崇科学对年，国人更是对这套方法深信不疑，
    但是，能够使用西方管理方法把企业管理好的也不见得其多。
      深究其原因，西方管理方法采用科学方法论，科学方法论的根旨确实量化管理，唯有工作量化，去除人性化的办法才是
    采用西方管理方法的基础。而我认为国内小企业所面临的都是人员少，资源少，时间短，也就是在资源比较短缺的情况下创业
    创业过程中，充满了人性化管理，简单举个小事例，西方管理中，委任一个项目，选择一个能够又匹配能力的人才即可，如果此人，能力不足
    那么在此过程中，西方管理，采用辞退换人，培训提供能力或者招聘能够承担此项能力的人即可。但是国内，由于资源受限，无法承担人员更换，
    短期培训带来的时间成本，资金成本，我们只能在资源受限的情况下，解决问题，因此回出现不合理原因，怕人临时走了，得哄着干活，能力不足，领导
    帮助承担，甚至还要应付工作人员性格带来的诸多问题，比如情绪化工作，职业素养低下。
       在国内企业中，尤其中小企业中，从业人员职业道德低劣，职业归属感底，职业修养也不高，难以将科学话管理提高到高效运行，并且在国内
    企业中，难以从价值认同感开始，管理企业，西方尤其注重权力与义务，更加注重荣誉感。而在国内，我们只注重权力。所以，国内企业的发展，都仅仅伴随着
    权力，人脉等等，缺少努力的结果，更加缺少努力，拼搏的精神。
       相较于我们的邻国日本，一直学习中国传统文化，忠恕精神，尤其是衍生出的武士道精神，也慢慢渗透到日本企业管理中，我们从稻盛和夫的著作中，看出，
    稻盛和夫尤其注重王阳明的心经，在企业管理中，尤其注重员工的归属感培养，也更加清楚归属感来源于企业领导人的人格魅力，通过这样的精神培养出来的性格，
    尤其非常明显的服从与执行。又加之，日本近代采用西方科学管理方法，不难看出，日本的企业管理的高效之处。

  ```

* 可执行方案要求
  ```
      可执行方案应该包含如下部分：
      1. 项目描述（为什么做这个项目）
      2. 项目目的（项目要达到什么样的目的）
      3. 项目时间要求（分几段时间）
      4. 项目阶段（各个阶段应该包含完成时间，可交付物，参与人员，验收标准）
      5. 验收标准
      6. 项目组织成员
      7. 项目成员责任描述
      8. 项目管理过程（定期周会，项目总结回，进度汇报）
      9. 人员考核标准 （定期汇报工作进度，如期按时包质量的完成工作）
      10. 人员加班问题
      11. 奖惩管理办法
      12. 奖金方法办法
      13. 项目沟通办法（积极通过邮件形式，沟通项目相关问题，积极参与并且提出建设性意见）
      14. 项目风险管理(人员流失，设备未能如期到位，资金未能如期到位，外部事件打断)

* 需求协议
  ```
        本着友好协商的原则，积极达成共同实现我们所一致认同的目标。
    我们一致认同的目标具体包含如下：
    1.我们达成一致，对所有目标能够在___年___月___日以内完成。
    2.我们达成一致，目标应该能够满足以下功能：
      ·······
      为了能够更好的完成我们所达成的一致目标，我们约定在完成目标过程中，
      1.每个工作周，汇报工作一次，洽谈一次。

          为了能够更好的完成我们所达成的一致目标，我们讲目标采用科学管理手段，将目标分拆，
      并且设定完成目标日期。具体目标分拆细节如下：
      目标1：
      1，目标中要有完成日期
      2，目标中要有完成目标要求
      3，目标中有干系人
      4，目标中有校验人
      5，目标要有+-日期（允许有调整日期）






  ```

* Git操作

  ```
    找到你要提交的目录下
    git init

    添加提交的远程仓库:
    码云:
    git remote add origin git@gitee.com:menglanyingfei/matlablearning.git
    github:
    git remote add origin git@github.com:menglanyingfei/matlablearning.git

    push前先将远程repository的修改pull下来
    git pull origin master

    作出修改, 如: 修改文件和增加文件(或者文件夹)

    git add .
    # 将添加的提交到列表上

    git commit -m 'first commit'
    # 提交你的文件, 添加消息”第一次提交”

    git push origin master
    #发送你的提交到GitHub

    成功提交了!

    git status
    # 查看git状态
